About me

My first steps towards game development were learning how to program. I learned the fundamentals of programming in a robotics class in middle school. I learned the foundation of C# a year later attempting my first prototype in Unity. I kept building small prototypes, and then took a computer science class where I learned Java in my last year of high school. Finally, I formally learned Java, C++, & C# at Lonestar College.


Beyond programming, I’ve also taken Graphic Design & Animation classes at the high school & college level respectively, as well as learning basic 3D modeling with Blender. I learned how to do all of these things because I wanted to understand at least some of the work that goes into every aspect of the creation of my favorite form of art.

My Projects


Boss Fight Prototype

Process - I first reviewed the code in the project I was editing & adding my own on to, then created a state machine to guide each action the boss character could do. First, I added the idle state, using it as a transition phase between the other states in the state machine. Then I tested the distance between the player & the boss during its animations so when the boss transitioned to its attacks, it would be in range. Then I added in the attacks & the fight was basically done. I did not create the scripts for the player, the art assets, or the animations. All of the programming for the boss was done by me however.

Tools Used - Unity

Challenges - This prototype was my first time using a state machine & caused me to be lost at the start on how to accomplish what I normally would using spaghetti code. After playing with the state machine for a bit & testing how to set up transitions & what to use transitions for, I learned how helpful state machines can be for enemies & players.


Stealth Game Prototype

Process - I started this project by creating a basic area to test the mechanics I was adding. The area included a blue capsule for the player, a red capsule to act as an enemy, two yellow spheres to act as patrol points, two walls the player could hide behind, & a plane to act as the ground. I imported a script for the player & set up a character controller to get them working quickly, then started working on the enemy. I created a state machine so the enemy could transition between patrolling from point to point, investigating when they heard a noise, & chasing the player when they saw them. While doing this I attached a navmesh agent to the enemy so they could easily move around obstacles in the area. The enemy uses a ray pointing in the direction of the player to detect walls between it & the player. If there are none, then the guard is capable of hearing the player. If the ray is unobstructed & the player is in front of the guard, then they will actively start chasing.

Tools Used - Unity

Challenges - The two challenges I faced during the creation of this project were learning how to create dot products & learning how to give the enemy a basic memory of the player’s last position. I solved the first issue by learning more about how dot products work online, & for the second issue I had an invisible game object follow the player when the guard could see or hear them, & then had the game object stop moving when the guard lost line of sight.
